## 核心修复方案

### P1: 双重锁修复

**当前问题**：
- `handleAlertState` 获取分片锁
- `handleTaskScheduling` 再次获取同一分片锁

**修复方案**：
```go
func (tw *TimeWheel) executeTask(node *taskSlot, now time.Time, currentSlot int) {
    // ... 任务执行 ...

    // 一次性获取锁，完成所有状态更新
    shard := tw.getShard(taskInfo.ID)
    shard.mu.Lock()

    tw.handleAlertStateLocked(node, taskInfo, result, now)
    tw.handleTaskSchedulingLocked(node, taskInfo, now, currentSlot)

    shard.mu.Unlock()
}
```

### P3: 异步持久化

**修复方案**：
```go
type AlertHistoryManager struct {
    recordCh chan AlertHistory
    stopCh   chan struct{}
    db       *sql.DB
}

func (m *AlertHistoryManager) Record(record AlertHistory) {
    select {
    case m.recordCh <- record:
    default:
        // 通道满，记录警告
    }
}

func (m *AlertHistoryManager) runBatchWriter() {
    batch := make([]AlertHistory, 0, 100)
    ticker := time.NewTicker(5 * time.Second)

    for {
        select {
        case record := <-m.recordCh:
            batch = append(batch, record)
            if len(batch) >= 100 {
                m.flushBatch(batch)
                batch = batch[:0]
            }
        case <-ticker.C:
            if len(batch) > 0 {
                m.flushBatch(batch)
                batch = batch[:0]
            }
        case <-m.stopCh:
            if len(batch) > 0 {
                m.flushBatch(batch)
            }
            return
        }
    }
}
```

## 使用示例

```go
// 创建时间轮，启用 SQLite 持久化
tw, err := timewheel.New(
    timewheel.WithSlotNum(60),
    timewheel.WithInterval(100*time.Millisecond),
    timewheel.WithSQLiteStore("./data/timewheel.db"),
    timewheel.WithAutoRestore(true),
)

// 启动（自动从 SQLite 恢复任务）
tw.Start()
defer tw.Stop()

// 启动健康检查服务
http.Handle("/", tw.HTTPHandler())
http.ListenAndServe(":8080", nil)
```
